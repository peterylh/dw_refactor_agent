# Schema Identity 与 DDL 演进设计

## 场景与问题

仅比较名称无法可靠判断 DDL 变化。一个字段从 `unit_price` 改为 `price_unit`，可能是重命名，也可能是删除旧字段后新增语义不同的字段。用名称相似度或注释猜测会让自动生成的 ALTER SQL 具有破坏性。

受管 DDL 因此为表和字段保存独立于名称的稳定 UUID。DDL 推导只在身份明确时识别重命名。

## 核心概念

- **`table_id`**：一张逻辑表在其生命周期内的稳定身份。
- **`column_id`**：一个字段在同一逻辑表内演进时的稳定身份。
- **受管 DDL**：项目配置的 ODS、MID 和 ADS DDL 目录中的 SQL 文件。
- **属性变化**：类型、nullable、default、comment、key 等不改变逻辑身份的变化。
- **替换**：旧对象消失、另一个新身份对象出现，不视为重命名。

ID 以紧邻表或字段定义的 SQL 注释保存，不写入 Doris 业务 COMMENT，也不影响实际表结构。

## 身份生命周期

| 操作 | `table_id` | `column_id` |
| --- | --- | --- |
| 新建表 | 新建 | 所有字段新建 |
| 表重命名 | 保留 | 全部保留 |
| 新增字段 | 保留 | 新字段新建 |
| 字段重命名或修改属性 | 保留 | 保留 |
| 复制、拆分、合并或替换 | 新建 | 新对象新建 |
| 删除名称后重新使用该名称 | 不复用 | 不复用 |

ID 表示 schema 对象身份，不表示跨表业务语义。两个表中含义相同的字段也不能共用 `column_id`。

## DDL 推导算法

推导前先解析基线和当前 DDL，并对受管项目执行严格校验。核心匹配逻辑如下：

```text
1. 用 table_id 匹配基线表与当前表；
2. 同一 table_id 名称变化 => 表重命名；
3. 在已匹配表内用 column_id 匹配字段；
4. 同一 column_id 名称变化 => 字段重命名；
5. 同一 identity 的可变属性变化 => ALTER/MODIFY；
6. 只在一侧出现的 identity => ADD 或 DROP；
7. 重命名并修改属性时，先重命名，再按最终名称修改。
```

不同 ID 即使名称、类型和注释很相似，也按删除和新增处理。Jaccard 或语义相似度只允许在显式 legacy 模式下处理没有 identity 的外部/历史 DDL，不用于项目重构主流程。

字段级 SQL 的稳定顺序为先删除、再重命名、最后新增和修改，避免后续语句引用尚未就绪的名称。大小写变化同样按 Doris 标识符规则处理，必要时使用安全的临时名称完成两阶段 rename。

## 校验规则

严格校验拒绝以下情况：

- 表或字段缺少 ID；
- ID 不是合法 UUID4；
- 同一对象关联多个 marker；
- marker 没有关联到可解析的表或字段；
- 表 ID 或字段 ID 在受管项目间重复；
- DDL 无法完整解析。

全局唯一性检查会扫描所有已配置受管项目，避免从另一个项目复制 DDL 后意外复用身份。推导器本身只读，不在分析过程中自动补 ID；ID 必须由显式 authoring 命令产生并进入代码审查。

## 与重构验证的关系

Schema identity 同时解决三个问题：

1. DDL deriver 能可靠生成表/字段 rename；
2. 语义分析能证明“仅名称变化”而不是对象替换；
3. Compare 能按 `column_id` 建立生产旧字段与 QA 新字段的一一投影。

如果用户声明重命名前后语义等价，但 identity 不完整或字段无法一一映射，验证必须 blocked，不能只比较公共列后宣称通过。

基线 DDL 是一次 refactor run 的冻结事实。资产布局、identity 规则或解析语义发生变化后，应创建新 run，不迁移旧 run 中的身份含义。

## 与血缘 ID 的区别

DDL UUID 是跨版本稳定的 schema identity。血缘库存储的表/字段整数 ID 是某次快照内的数据库 surrogate key，两者生命周期不同，不能互换。

血缘匹配仍使用规范化表/字段名称；需要跨重命名前后关联时，由重构模块显式使用 DDL identity mapping。

## 设计边界

- Identity 只证明对象连续性，不证明 SQL 结果语义等价。
- ID 不自动写入 Doris，也不依赖生产库元数据。
- 普通开发操作必须保留已有 ID；复制资产时必须生成新 ID。
- 详细操作步骤由重构指南和 CLI help 维护，本文只描述身份语义和推导原则。
